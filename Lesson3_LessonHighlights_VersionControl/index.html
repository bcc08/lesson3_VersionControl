Highlight #1 Version Control

Version control is a very useful tool for group projects in which multiple files have to be edited. These files can be accessed simultaneously or at the same time. What it does is allow the user to keep up with every version of a file that was created or edited. For instance say I edited some code earlier today and my project partner decides to add more to the file that I edited earlier. What version control does is keep track of every version of that file just in case something goes wrong. If the code was working fine after I finished editing and doesn’t work after my partner edits, version control allows you to retrieve the old file and also see the changes that were made on the new file.

Highlight #2 Git

Git is a VCS that is used to store and process information. Git is unique because it handles the information differently than other VCS. Instead of processing the files for changed data Git process data information by keeping snapshots of previously saved files. If the new data has the same snapshot, meaning it’s unchanged; GIT does not save the file again. Instead it gives a link to the previous file already saved. One thing that is very likeable about Git is that it is considered to have integrity. This is said because Git does not allow data of a file or directory to be changed without it knowing

Highlight #3 Tracked/Untracked Repository Files

When working on files in the directory they can either be in an “untracked” or a “tracked” state. If a file is tracked that means it has already been identified in a previous snapshot. This does not necessarily mean that the file has not been modified as the file can be staged, modified, or unmodified. Untracked files are files that were not included in a previous snapshot and are not within the staging area. All files are tracked when you first clone a repository. These files are considered unmodified until they are edited. The files only become modified when they are edited after the last commit.

Highlight #4 Checking Status Of GIT Files

Git has a command tool that allows the user to determine the state of files. The command to determine the status is “git status.” If no files appear, whether tracked or modified, that means you have a clean directory. This Git command does however recognize untracked files. Once the command has been entered it lists the untracked files under the “untracked files” header. It also gives you the current branch location as well. All untracked files are left untracked until you specifically tell it to be tracked. This is done to help prevent the user from including files that were not meant to be included.

Highlight #5 GitHub

GitHub was found February 8, 2008. As of 2015 GitHub has more than 11 million users and 29.4 million repositories. Therefore within almost the last 8 years GitHub has become the largest host of source code in the world. It is unique because it is not strictly a command tool like Git. GitHub offers a graphical web based interface along with mobile integration. Other key features that it offers consist of access control, wikis for each project, bug trafficking, task management, and feature request. When creating a GitHub account you can choose from paid private repositories or free accounts for open source projects.
